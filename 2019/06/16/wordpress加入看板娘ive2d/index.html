wordpress美化之看板娘live2d

pan · 2019-6-16 · 次阅读

wordpress主题页面展示插入看板娘live2D

效果展示

下载

https://github.com/panpeihui/live2/tree/master/live2d

  • 把live2d.zip包上传解压到wordpress/wp-content/themes/安装的主题目录/

解压

提取以下文件及目录
assets
demo.html
live2d
mv .* /wordpress/wp-content/themes/安装的主题目录/

修改文件

vim assets/waifu-tips.js
233     loadlive2d('live2d', './live2d/get/?id='+modelId+'-'+modelTextur
    esId, console.log('live2d','模型 '+modelId+'-'+modelTexturesId+' 加>
    载完成'));
244         url: './live2d/'+modelTexturesRandMode+'_textures/?id='+mode
    lId+'-'+modelTexturesId,
264         url: './live2d/'+modelTexturesRandMode+'/?id='+modelId,

把./live2d修改成绝对路径，不然会报错
列如：/wp-content/themes/xxx/assets/live2d XXX为主题文件目录

 vim demo.html
<script type="text/javascript" src="./assets/jquery.min.js"></script>
<script type="text/javascript" src="./assets/waifu-tips.js"></script>
<script type="text/javascript" src="./assets/live2d.js"></script>

把 assets也修改成绝对路径 列如：/wp-content/themes/xxx/assets
这样就可以访问demo.html检查看板娘是否可以正常食用
https://hostname/wp-content/themes/xxx/demo.html
结合浏览器F12检查报错信息进行修改，一般是路径问题

  • 接下来就可以把demo.html代码复制到主题安装目录下的header.php

html插入代码

head标签

vim header.php
<head>
    <meta http-equiv="Content-Type" content="text/html; charset=UTF-8" />  
    <link rel="stylesheet" type="text/css" href="/wp-content/themes/xxx/assets/waifu.css">
    <script src="/wp-content/themes/xxx/assets/jquery.min.js"></script>
</head>

boby标签

# vim footer.php

<script type="text/javascript" src="/wp-content/themes/xxx/assets/waifu-tips.js"></script>

这样就可以在网站主题里显示看板娘live2d了

还有另一种方法

插件方式安装

https://github.com/panpeihui/Poilive2d 下载插件不需要解压

报错提示

在网站后台进行插件安装会提示：
The uploaded file exceeds the upload_max_filesize directive in php.ini

报错解决

find / -type f -name "php.ini"
vim /xxx/php.ini
upload_max_filesize = 2M  例如：修改成32M
post_max_size = 8M        例如：修改成32M  
#/etc/init.d/httpd restart

然后在进行安装插件，刷新网页就Okey了

安装插件及主题提示Ftp或者权限问题

解决方法：

cd /wordpress
cd /wp-content/themes/
mkidr tmp
chmod -R 777 tmp 
chmod -R 777 wp-content
vim wp-config.php   在后面插入内容
define('WP_TEMP_DIR', ABSPATH.'wp-content/tmp');
define("FS_METHOD", "direct");
define("FS_CHMOD_DIR", 0777);
define("FS_CHMOD_FILE", 0777);

这样就完美解决权限问题，食用流畅
